# Project 2: Full-Stack Application — Rubric & Requirements

**Weight:** 20% of final grade | **Total Points:** 200

---

## Functional Requirements

- [ ] Complete full-stack application (frontend + backend + database)
- [ ] User authentication (JWT or OAuth)
- [ ] 3+ distinct features/roles
- [ ] Real-time updates OR complex state management
- [ ] Public API with documentation
- [ ] Professional UI/UX

## Technical Requirements

- [ ] **AI Modalities:** Use the two AI modalities appropriately (Claude Web + IDE-Centric)
- [ ] **Tech Stack:** React/Next.js + Tailwind CSS, Node.js/Express or Next.js API routes, PostgreSQL or MongoDB
- [ ] **Testing:** 80%+ test coverage (unit + integration + E2E)
- [ ] **Evaluation Suite:** Automated tests, code quality metrics, security scanning
- [ ] **CI/CD:** Advanced multi-stage pipeline, deploy previews, coverage reporting

## Agile Process

- [ ] 2+ documented sprints
- [ ] Sprint planning documents and retrospectives
- [ ] User stories with clear acceptance criteria

## Grading Rubric (200 Points)

### Functionality — 45 pts

| Level | Points | Criteria |
|---|---|---|
| Excellent | 45 | All features complete, auth working perfectly, API functional and well-designed, polished UI/UX |
| Good | 36 | Most features complete, authentication works, API functional, good UI/UX |
| Satisfactory | 27 | Core features work, basic authentication, API works, acceptable UI/UX |
| Needs Improvement | 18 | Some features missing, auth issues, API problems, poor UI/UX |
| Unsatisfactory | 0 | Major features broken or missing |

### Technical Excellence — 60 pts

| Level | Points | Criteria |
|---|---|---|
| Excellent | 60 | Outstanding architecture, 80%+ test coverage, excellent DB design, strong security |
| Good | 48 | Good architecture, 70–79% coverage, good DB design, adequate security |
| Satisfactory | 36 | Acceptable architecture, 60–69% coverage, basic DB design, some security |
| Needs Improvement | 24 | Weak architecture, <60% coverage, poor DB design, security gaps |
| Unsatisfactory | 0 | No clear architecture, minimal tests, major security issues |
